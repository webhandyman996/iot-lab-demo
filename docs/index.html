資料串接架構：從感測器到雲端儀表板（Sensor to Dashboard Pipeline）

這門課示範的完整資料流程是：感測器量到數值 → 開發板整理並連網送出 → Google表單自動收集 → 試算表即時累積資料 → Looker Studio 儀表板自動畫圖顯示。整條路徑完全不用自己架伺服器。

Looker Studio
儀表板

連接試算表為資料來源，圖表自動更新

  1. 感測器 → MCU：感測器透過腳位（GPIO）或 I2C 匯流排把量到的原始數值傳給開發板。如果主控板本身沒有WiFi功能（例如 Arduino Mega2560），會再透過序列埠（UART）把資料交給旁邊負責連網的 ESP32/ESP8266。
  2. MCU → Google表單：裝置連上WiFi後，不需要真的打開表單網頁填寫，而是直接對表單的固定網址（.../formResponse）發送 HTTP POST 請求，把每個感測器數值放進對應的 entry.XXXXXXX 參數裡，效果等同於「自動幫你把表單填好送出」。這個 entry 代碼可以在表單編輯畫面按「預覽」，再按右鍵「檢查」原始碼，找到每個問題輸入框的 name="entry.XXXXXXX" 屬性取得。
  3. Google表單 → 試算表：Google表單本身就會自動把每一筆送出的回覆存成試算表的一列，等於免費幫你架好一個會即時更新的資料庫，完全不用自己寫後端程式或資料庫。
  4. 試算表 → Looker Studio：把這份試算表設定成 Looker Studio（原 Google Data Studio）的資料來源，畫面上的折線圖、圓餅圖等會自動抓最新的資料列重新繪製，就變成一個會自動更新的監控儀表板——本 repo 裡的 Dashboard/factory_monitor_dark.json 就是這種儀表板的版面設定範例（智慧工廠冷卻水塔監控主題）。

跟 Board2Web 範例的做法比較

Google表單方案本repo Board2Web 方案（Apps Script）
資料接收端Google表單（免寫程式）自己寫的 Google Apps Script Web App
驗證/安全性無內建驗證，任何人都能送資料可自訂 API_KEY 驗證，較安全
資料處理彈性低，只能收資料高，送出前可先做邏輯判斷、格式轉換
建置難度最簡單，適合快速做出能動的demo需要寫 Apps Script 程式碼
最終資料去處Google試算表Google試算表
兩種做法殊途同歸，最後都是把資料存進 Google試算表，再交給 Looker Studio 畫圖。差別只在「怎麼把資料送進試算表」這一段：表單免寫程式但沒有驗證，Apps Script 較安全但要多寫一點程式碼。
